# Contributing

Thanks for your interest in contributing! Below are the basic guidelines and recommendations for contributors.

1. Before you start, open an issue describing what you plan to change or add.
2. Fork the repository and create a branch with a clear name, e.g. `feature/description` or `fix/description`.
3. Follow the existing code style. For Python, follow PEP8.
4. In your pull request, describe what changed and why. Include steps to verify the change.
5. If your change affects any public API or data structures, describe it in the PR and update `README.md`.

Quick checks before committing:
- Run `npm run build` in `frontend` (if you changed frontend code).
- Ensure the backend installs with `python -m pip install -e ./backend`.

Thanks — your contribution is appreciated!
